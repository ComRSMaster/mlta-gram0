# Интерпретатор грамматик класса 0

Математическая логика 2026, группа Б05-515, Хохлов Владислав

## Описание

**Грамматика класса 0** - набор из $\Sigma$, $\Gamma$, $P$, $S$, где:

* **Терминалы ($\Sigma$)** - символы, из которых состоят итоговые слова.
* **Нетерминалы ($\Gamma$)** - технические символы, которые нужны как вспомогательные для обработки данных.
* **Правила ($P$)** - список замен вида $\alpha \to \beta$: подстрока $\alpha$ заменяется на строку $\beta$. В $\alpha$ должен быть хотя бы один нетерминал.
* **Начальная строка ($S$)** - программа начинает применять правила ($P$) к этой строке $S$.

**Вывод** - последовательность строк, полученных из $S$ последовательным применением правил $\alpha \to \beta$.

Вывод считается **оконченным**, когда в строке отсутствуют нетерминалы. Для этого сделано ограничение в правилах ($P$) - в $\alpha$ должен быть хотя бы один нетерминал.

Грамматика **вычисляет строковую функцию** `f(x)` от аргумента `x`, если в ней можно из `B x E` вывести `b f(x) e`, где `B`, `E`, `b`, `e` - нетерминалы для маркировки начала и конца строки. Будем считать, что вычисление функции `f(x)` **окончено**, когда для текущей строки не применимо ни одно правило из $P$. Если вычисление `f(x)` **окончено** и текущая строка выглядит как `b ... e` (первый и последний символы - нетерминалы для маркировки начала и конца строки) и в `...` не осталось нетерминалов, то значение `f(x)` **определено** и равно этому `...`, иначе **не определено**.

Такие грамматики не детерминированы, поэтому для того, чтобы наша грамматика была детерминированной, интерпретатор будет выбирать первое применимое правило $\alpha \to \beta$ в порядке их перечесления в $P$ и заменять в текущей строке первое вхождение $\alpha$ на $\beta$.

## Синтаксис

Каждый **символ** в грамматике - строка `s`, состоящая из строчных и заглавых латинских букв, цифр и символов `` !"#$%&'()*+,-./:;<=>?@[\]^_`{|}~ ``. Символ **не может начинаться с решетки (`#`)**, также символ **не может быть стрелкой (`->`)**.

**Строка** в грамматике - последовательность этих символов, разделенных пробелом.

**Нетерминальные символы ($\Sigma$)** - символы, которые начинаются с точки (`.s`), в частности символ, состоящий только из точки тоже нетерминальный.

**Терминальные символы ($\Gamma$)** - символы, которые начинаются не с точки (просто `s` без точки в начале).

**Начальная строка ($S$)** - `.B ... .E`, где `.B`, `.E` - нетерминалы для маркировки начала и конца строки. `...` берется либо из файла, либо из стандартного ввода и к нему автоматически дописываются `.B` в начало и `.E` в конец.

Нужно описать **список правил ($P$)** в файле (рекомендуемое расширение файла `.g0`) в следующем формате:

В файле каждая строка либо пустая, либо комментарий (должен стоять символ решетки, а после него может идти любая последовательность символов), либо само правило $\alpha \to \beta$ - заменить подстроку `` a₀ a₁ a₂ ... aₙ `` на `` b₀ b₁ b₂ ... bₘ ``, пробелы отделяют разные символы и зарезервированный символ стрелки:
```text
a₀ a₁ a₂ ... aₙ -> b₀ b₁ b₂ ... bₘ
```

## Сборка

```shell
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target gram
```

Исполняемый файл `gram` появится в папке `bin`.

## Использование

```
gram program.g0 [-v] [-i input_file] [-o output_file]
```

Если задан флаг `-v` (verbose), то весь процесс вывода (что на что заменяется и по какому правилу) будет выведен в `stderr`.

Если задан флаг `-i` (input), то `x` (аргумент функции) будет считан из файла `input_file`. Иначе он будет считан из стандартного ввода `stdin`. Начальная строка $S$ = `.B x .E`, которая пойдёт в грамматику образуется прибалением `.B` в начало и `.E` в конец к `x`.

Если значение `f(x)` **определено**, то есть последняя строка в выводе выглядит как `.b f(x) .e`, то её значение будет выведено (без `.b` и `.e` в начале и в конце) в таком формате: если задан флаг `-o` (output), то значение `f(x)` будет выведено в файл `output_file`. Иначе оно будет выведено в стандартный вывод `stdout`.

```shell
cmake -B build -DBUILD_TESTING=ON
```

```shell
cmake --build build --target unit_tests
bin/unit_tests
```

#### 4. Запуск Integration-тестов:
```shell
cmake --build build --target integration_tests
```
